Vigilar — FAQ

Common questions about Vigilar's privacy, model, platform support, and pricing.

Does Vigilar send my data to the cloud?

No. Everything runs on your device. The AI model runs locally. Breach checks use k-anonymity (only first 5 chars of SHA-1 hash sent).

What model does Vigilar use?

Ember Prime v8c, a 12B parameter model quantized to 4-bit (7.4GB). It runs via llama.cpp on desktop and MLC on mobile.

How much RAM do I need?

16GB minimum for desktop (model uses ~8GB). 8GB minimum on mobile (smaller quantization).

Does it work offline?

Yes, fully offline after installation. Breach checks need internet (HIBP API), but everything else works offline.

Is it open source?

The app source is on GitHub (github.com/SEC-APAC-MB/vigilar-app). The model is built on Apache 2.0 licensed components.

What platforms are supported?

macOS (Apple Silicon, DMG installer) and Android (APK). iOS planned via App Store.

How does the ARCZT Vault link work?

Deep-link bridge. Vigilar sends breach metadata (domain, name, date, severity) to ARCZT. No credentials are shared. The two apps are separate products.

Can I use Vigilar without ARCZT Vault?

Yes. Vigilar works standalone. ARCZT integration is optional.

What's the difference between Free and Pro?

Free includes all security features (dashboard, breach monitor, threat detection, security score). Pro adds AI Companion Chat, priority breach alerts, and identity theft protection.

Is there a family plan?

Yes. €19.99/month covers 5 devices with shared breach monitoring and a family security dashboard.
