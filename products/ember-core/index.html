What is Ember Core?

Ember Core is a custom-trained 12B parameter language model built from Apache-2.0 sources. It is quantized to 4-bit (Q4_K_M, 7.4GB) and runs entirely on-device — no cloud inference, no API calls, no network dependency.

Model Lineage

  • Base: Gemma 4 12B skeleton (Apache 2.0)
  • SFT: Supervised fine-tuning on curated security + conversation data
  • DPO: Direct preference optimization for safety and helpfulness alignment
  • Edge LoRA: Edge-specific adaptation for on-device performance
  • Governance LoRA: Anti-identity, redline enforcement, safety guardrails
  • Fingerprint: Model fingerprinting for provenance verification

Benchmarks

Humanity's Last Exam (HLE) — 3-way benchmark:

  • Base architecture (untrained): 11/100
  • Edge (no governance): 17/100
  • Ember Prime (+governance+fingerprint): 24/100
  • Total improvement: +13 percentage points, zero regression

Technical Specs

  • Parameters: 12B
  • Quantization: Q4_K_M (4-bit, 7.4GB on disk)
  • Inference: llama.cpp (desktop), MLC (mobile)
  • Context window: 4096 tokens
  • Training data: 4-pillar curriculum (security, conversation, reasoning, governance)
  • License: Apache 2.0 components, assembled and trained by SEC-APAC-MB
